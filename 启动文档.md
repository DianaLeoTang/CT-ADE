# CT-ADE 工程介绍与启动步骤

## 一、工程介绍

### 1. 项目定位

**CT-ADE**（Clinical Trial - Adverse Drug Event）是一个**基于临床试验结果的不良药物事件（ADE）预测与评估基准**，对应论文：

- *An Evaluation Benchmark for Adverse Drug Event Prediction from Clinical Trial Results*  
- 发表于 *Scientific Data*（2025），Nature Publishing Group

### 2. 核心功能

- **数据流水线**：从 ClinicalTrials.gov、PubChem、DrugBank、ChEMBL 等来源采集与清洗数据，经 MedDRA 标准化后生成 **CT-ADE-SOC**（系统器官分类）和 **CT-ADE-PT**（首选术语）两类基准数据集。  
- **建模**：支持两类模型训练与评估：  
  - **DLLMs**（判别式大语言模型）：二分类等任务。  
  - **GLLMs**（生成式大语言模型）：支持 Llama3、OpenBioLLM、Meditron 等基座，可做 S / SG / SGE 等配置。

### 3. 仓库结构概览

| 层级 | 说明 |
|------|------|
| **a0 / a1 / a2** | 下载临床试验、提取已完成/终止试验结果、单药试验预处理 |
| **b0 / b1** | 下载 PubChem CID 及详情 |
| **c0** | 从 DrugBank 提取 DBID 详情 |
| **d0 / e0** | 从 ChEMBL 提取批准药与 USAN 详情 |
| **f0** | 构建统一化学数据库（PubChem + DrugBank + ChEMBL） |
| **g0 ~ g3** | 生成原始 CT-ADE、MedDRA 标注、分类数据集、友好标签版本 |
| **data/** | MedDRA、DrugBank、clinicaltrials_gov、pubchem、chembl_* 等数据目录 |
| **modeling/DLLMs** | 判别式模型配置与训练 |
| **modeling/GLLMs** | 生成式模型配置与训练（S/SG/SGE） |
| **src/** | 公共逻辑（如 `meddra_graph.py`） |

### 4. 开发环境

- **推荐**：Ubuntu 22.04.3 LTS，x86_64  
- **Python**：3.10.12  
- **依赖**：见 `requirements.txt`；`transformers`、`trl` 需从各自 Git 仓库安装**开发版**。

## 三、启动步骤（两种方式）

### 方式 A：从检查点复现（推荐，数据已就绪时）

在完成「环境准备」且已放置 MedDRA、DrugBank 和 HuggingFace 检查点后，在项目根目录执行：

```bash
# 3. 从 DrugBank 提取 DBID 详情
python c0_extract_drugbank_dbid_details.py

# 4. 创建统一化学数据库
python f0_create_unified_chemical_database.py

# 5. 生成原始 CT-ADE 数据集
python g0_create_ct_ade_raw.py

# 6. 生成 MedDRA 标注
python g1_create_ct_ade_meddra.py

# 7. 生成分类数据集（CT-ADE-SOC / CT-ADE-PT）
python g2_create_ct_ade_classification_datasets.py

# 8. （可选）生成用户友好标签版本
python g3_create_ct_ade_friendly_labels.py
```

以上顺序不可打乱，按 3→4→5→6→7→8 依次执行即可。

### 方式 B：从零构建完整流水线

若没有检查点，需要先跑完 a/b/c/d/e 再跑 f、g：

```bash
# 临床试验
python a0_download_clinical_trials.py
python a1_extract_completed_or_terminated_interventional_results_clinical_trials.py
python a2_extract_and_preprocess_monopharmacy_clinical_trials.py

# PubChem
python b0_download_pubchem_cids.py
python b1_download_pubchem_cid_details.py

# DrugBank / ChEMBL
python c0_extract_drugbank_dbid_details.py
python d0_extract_chembl_approved_CHEMBL_details.py
python e0_extract_chembl_usan_CHEMBL_details.py

# 统一库与 CT-ADE 生成（与方式 A 相同）
python f0_create_unified_chemical_database.py
python g0_create_ct_ade_raw.py
python g1_create_ct_ade_meddra.py
python g2_create_ct_ade_classification_datasets.py
python g3_create_ct_ade_friendly_labels.py  # 可选
```

## 五、模型训练

### 判别式模型（DLLMs）

```bash
cd modeling/DLLMs
# 按需编辑 config.py 后执行
export CUDA_VISIBLE_DEVICES="0"
export MIXED_PRECISION="bf16"
FIRST_GPU=$(echo $CUDA_VISIBLE_DEVICES | cut -d ',' -f 1)
BASE_PORT=29500
PORT=$(( BASE_PORT + FIRST_GPU ))
accelerate launch --mixed_precision=$MIXED_PRECISION \
  --num_processes=1 --num_machines=1 --dynamo_backend=no \
  --main_process_port=$PORT \
  train.py
```

多卡时把 `CUDA_VISIBLE_DEVICES` 设为如 `"0,1,2,3"`，并相应增加 `--num_processes`。

### 生成式模型（GLLMs）

```bash
cd modeling/GLLMs
# 将 config-llama3.py / config-meditron.py / config-openbiollm.py 之一复制为 config.py 并修改
python train_SGE.py
```

## 六、快速启动检查清单

1. 使用 **Python 3.10**，安装 `requirements.txt` 及 transformers/trl 开发版。  
2. 将 **MedDRA** 放到 `./data/MedDRA_25_0_English`，**DrugBank XML** 放到 `./data/drugbank`。  
3. 若从检查点复现：下载 **CTADE_v1_initial_release_checkpoint** 中四个目录到 `./data/`。  
4. 在项目根目录按顺序执行：`c0` → `f0` → `g0` → `g1` → `g2`（可选 `g3`）。  
5. 训练时进入 `modeling/DLLMs` 或 `modeling/GLLMs`，按上述命令启动。

按上述步骤即可完成从环境搭建、数据复现到模型训练的完整启动流程。若你告诉我当前卡在哪一步（例如没有 DrugBank、或只想跑 GLLMs），可以再给你写一版最小可运行清单或对应命令。
